Retrieval-Augmented Generation

Memory on demand — augmenting a language model with a searchable knowledge base

A language model's knowledge is frozen at training time. Ask it about yesterday's news, your company's internal docs, or a paper published last week, and it draws a blank. Retrieval-Augmented Generation (RAG) solves this by giving the model a searchable external memory — a vector database where documents live as points in high-dimensional space, and the most relevant ones are fetched on demand to ground every response.

A vector database is a library where the shelves are organised by meaning, not alphabet. Similar ideas are geometrically nearby. RAG gives an LLM a dynamic, updatable external memory — add a new document, and the model can immediately reason about it without retraining.
Chunking matters: Long documents are split into chunks of 500–1,000 tokens before embedding. Too large, and each chunk covers too many topics (low specificity). Too small, and context is lost. The chunk size is a critical hyperparameter in any RAG system.

Retrieval is a nearest-neighbour search in embedding space — meaning becomes geometry.

2. The RAG Pipeline

End-to-end retrieval-augmented generation

Click "Run pipeline" to animate the full flow, or step through one stage at a time.

1. Encode: q → embed(q) ∈ ℝd
2. Retrieve: top-k docs = argmaxd ∈ DB sim(embed(q), embed(d))
3. Augment: prompt = [system] + [retrieved docs] + [user query]
4. Generate: response = LLM(prompt) — grounded in retrieved evidence
Without RAG, a model's knowledge is frozen at training time. With RAG, it can reason over any document you give it — company policies, medical records, legal contracts. This is how enterprise AI systems access private knowledge without retraining a billion-parameter model.
Hallucination contrast: Without RAG, a model asked “What is our refund policy?” will confabulate a plausible-sounding answer from training data. With RAG, it retrieves the actual policy document and quotes from it. The grounding mechanism transforms an unreliable narrator into a citeable one.
Embedding drift: If you re-embed documents with a newer model version, the vectors change — and old query embeddings no longer align. Production RAG systems version their embedding models and re-index periodically.

RAG = search + synthesis — the pattern that makes LLMs useful in the real world.

Why This Matters

Every major enterprise AI deployment uses RAG. It is the bridge between a general-purpose model and domain-specific knowledge. A model that can retrieve is a model that can stay current, cite sources, and avoid hallucination — the three requirements for AI in production.

The mathematics is elegant: encode everything into vectors, search by geometry, and let the model synthesise a grounded answer. The same cosine similarity that finds relevant documents also powers recommendation engines, plagiarism detection, and semantic search across billions of web pages.
