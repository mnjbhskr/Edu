Linear Programming

●●●● Intermediate

Optimise a linear objective subject to linear constraints — solved visually

1. Introduction

Linear programming (LP) is a method for finding the best outcome — such as maximum profit or minimum cost — in a mathematical model whose requirements are represented by linear relationships. It is one of the most widely used techniques in operations research and applied mathematics.

The core idea is simple: you have a linear objective function that you wish to maximise or minimise, subject to a set of linear inequality constraints. The collection of all points satisfying every constraint forms a convex polygon called the feasible region, and the optimal solution always lies at one of its vertices.

Real-world applications include:

  • Production planning — deciding how many units of each product to manufacture given limited raw materials and labour
  • Resource allocation — distributing a budget across projects to maximise return
  • Diet problems — finding the cheapest combination of foods that meets all nutritional requirements
  • Transportation — minimising shipping costs while meeting supply and demand at every location
George Dantzig developed the simplex method in 1947 while working for the US Air Force. The technique proved so powerful that it transformed military logistics, airline scheduling, and manufacturing worldwide. Today, LP solvers handle problems with millions of variables.

2. Formulating the Problem

The Building Blocks

Every linear programme has four components:

1. Decision variables

The unknowns you control. In two dimensions these are x and y — for example, the number of chairs and tables a factory produces.

2. Objective function

A linear expression you wish to maximise or minimise:

Maximise (or minimise):   Z = c1x + c2y

3. Constraints (inequalities)

Linear inequalities that limit the decision variables:

a1x + b1y ≤ c1
a2x + b2y ≤ c2
…

4. Non-negativity conditions

x ≥ 0,   y ≥ 0

These ensure the solution makes physical sense (you cannot produce negative quantities).

Worked Example — Factory Problem

A furniture workshop makes chairs (x) and tables (y). Each chair requires 2 hours of woodwork and 1 hour of finishing. Each table requires 3 hours of woodwork and 2 hours of finishing. The workshop has at most 120 hours of woodwork and 80 hours of finishing available per week. Each chair earns £40 profit; each table earns £70.

Maximise: Z = 40x + 70y
Subject to:
  2x + 3y ≤ 120   (woodwork hours)
  x + 2y ≤ 80     (finishing hours)
  x ≥ 0,   y ≥ 0
The art of LP formulation is translating a real-world scenario into these four components. Once formulated correctly, the mathematics takes over — the graphical method (for two variables) or the simplex algorithm (for many variables) will find the optimum.

4. Finding the Optimal Solution

The Vertex (Corner Point) Theorem

The Fundamental Theorem of Linear Programming states: if a linear programme has an optimal solution, then at least one optimal solution occurs at a vertex (corner point) of the feasible region.

This means we only need to evaluate the objective function at each vertex and pick the best. For the factory problem above:

Vertexx (chairs)y (tables)Z = 40x + 70y
A — Origin 0 0 £0
B — x-axis ∩ woodwork 60 0 £2,400
C — woodwork ∩ finishing 24 24 £2,640 ☆
D — y-axis ∩ finishing 0 40 £2,800
▶ Wait — vertex D gives £2,800 > £2,640?

Check whether D = (0, 40) satisfies all constraints:

2(0) + 3(40) = 120 ≤ 120   ✓
0 + 2(40) = 80 ≤ 80   ✓

Indeed D is feasible and gives Z = £2,800. So the optimum is actually at D, not C. This reinforces why you must check every vertex — intuition alone is unreliable.

Optimal solution: make 0 chairs and 40 tables for a maximum profit of £2,800 per week.

The vertex theorem works because the objective function is linear: it cannot have interior maxima or minima on a convex set. A linear function on a convex polygon is maximised (and minimised) at a vertex. This is why LP is so powerful — you only need to check finitely many points.

6. Worked Examples

Example 1 — Production Planning

A bakery makes croissants (x) and sourdough loaves (y). Each croissant uses 100 g flour and 20 g butter. Each loaf uses 500 g flour and 10 g butter. The bakery has 10 kg flour and 1.2 kg butter per day. Profit is £1.50 per croissant and £3.00 per loaf.

Maximise: Z = 1.5x + 3y
Subject to:
  100x + 500y ≤ 10000  (flour, in grams)
  20x + 10y ≤ 1200    (butter, in grams)
  x ≥ 0,   y ≥ 0
▶ Show full solution

Simplify by dividing the flour constraint by 100 and butter by 10:

x + 5y ≤ 100   (flour)
2x + y ≤ 120    (butter)

Vertices of feasible region:

O = (0, 0)  → Z = 0
A = (60, 0)  → Z = 90
B = intersection of x + 5y = 100 and 2x + y = 120
   Solve: x = 100 − 5y  → 2(100 − 5y) + y = 120
   200 − 10y + y = 120  → 9y = 80  → y ≈ 8.89
   x = 100 − 5(8.89) ≈ 55.56
   B ≈ (55.56, 8.89)  → Z ≈ 110.00
C = (0, 20)  → Z = 60

Optimal: make approximately 55.6 croissants and 8.9 loaves for £110.00 daily profit. (In practice, round to whole numbers and check feasibility.)

Example 2 — Diet / Nutrition Problem

A dietitian wants to meet a patient’s minimum daily requirements of 60 mg vitamin C and 40 mg iron at minimum cost. Two supplements are available: Supplement A (x tablets) provides 10 mg vitamin C and 5 mg iron per tablet at £0.30 each. Supplement B (y tablets) provides 15 mg vitamin C and 8 mg iron per tablet at £0.50 each.

Minimise: Z = 0.30x + 0.50y
Subject to:
  10x + 15y ≥ 60  (vitamin C)
  5x + 8y ≥ 40    (iron)
  x ≥ 0,   y ≥ 0
▶ Show full solution

For ≥ constraints, the feasible region is above each line (away from the origin). We seek the vertex nearest the origin (minimising cost).

Vitamin C boundary: 10x + 15y = 60  → y = (60 − 10x)/15
Iron boundary: 5x + 8y = 40  → y = (40 − 5x)/8

Intersection of the two constraints:

10x + 15y = 60  and  5x + 8y = 40
Multiply second by 2: 10x + 16y = 80
Subtract first: y = 20
Substitute: 10x + 15(20) = 60  → 10x = −240  → x = −24

Since x = −24 violates x ≥ 0, the two constraint lines do not intersect in the feasible region. The relevant vertices are on the axes:

A: x-axis ∩ iron  → 5x = 40 → x = 8  (check vitamin C: 80 ≥ 60 ✓)
   A = (8, 0)  → Z = £2.40
B: y-axis ∩ iron  → 8y = 40 → y = 5  (check vitamin C: 75 ≥ 60 ✓)
   B = (0, 5)  → Z = £2.50

Optimal: buy 8 tablets of Supplement A and none of Supplement B for a minimum daily cost of £2.40.
