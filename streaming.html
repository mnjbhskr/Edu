How Streaming Works

The mathematics of video, music, and data over imperfect networks — from entropy to error correction

Imagine watching Netflix on a train with spotty WiFi, or listening to Spotify on the London Underground. How does your device deliver smooth, high-quality video or music despite dropped packets, electrical interference, and bandwidth constraints? The answer lies at the intersection of three mathematical fields: information theory, data compression, and error correction.

When you stream a film, the data travels as packets over imperfect networks. Along the way, some bits may flip due to interference. The sender compresses the data using Huffman coding to save bandwidth, breaks it into small chunks, adds error-correcting codes like Hamming codes to fix corrupted bits, and sends it all across the internet. The receiver reconstructs everything perfectly.

This journey is governed by elegant mathematics discovered in the 1940s by Claude Shannon, David Huffman, and others—ideas so powerful they're used in every streaming service, satellite communication, and data storage system in the world.

1. Information Theory & Entropy

Shannon's Insight
In 1948, Claude Shannon revolutionised our understanding of information. He asked: how many bits do you need to transmit a message? The answer depends not on the alphabet size, but on how surprising each symbol is. If you're sending English text, the letter 'e' is common and carries little surprise—it needs fewer bits. The letter 'z' is rare and surprising—it needs more bits.
Shannon's Entropy Formula:
H = −Σ p(x) log₂ p(x)
where p(x) is the probability of symbol x and the sum is over all symbols. H is measured in bits and represents the average information (or uncertainty) per symbol.

2. Huffman Coding

Variable-Length Encoding
David Huffman discovered an optimal algorithm for assigning variable-length binary codes to symbols based on their frequency. Common symbols get short codes (e.g., 'e' → "0"), rare symbols get longer codes (e.g., 'z' → "10110"). The algorithm builds a binary tree bottom-up, repeatedly merging the two smallest-frequency nodes until one tree remains.
Why It Works: Huffman coding achieves compression ratios close to Shannon entropy—it's theoretically optimal for single-symbol encoding. It's used in JPEG, PNG, MP3, ZIP, and countless other formats.

3. Beyond Huffman: Run-Length and Pattern Matching

Run-Length Encoding (RLE)
Before Huffman, a simpler method existed: if you see the same symbol repeated, replace it with the symbol and a count. For example, "AAABBA" becomes "3A2B1A". This is highly effective for images with large solid regions.
LZ77 and Sliding Windows
The LZ77 algorithm (used in DEFLATE, which powers ZIP and gzip) uses a sliding window to find repeated patterns anywhere in the recent history. When a pattern matches, it's encoded as a reference (offset, length) instead of literal bytes. This combines compression of both runs and repeated phrases.

4. Error-Correcting Codes: Hamming(7,4)

The Problem: Noise
During transmission over radio, copper cables, or optical fibres, bits occasionally flip due to electromagnetic interference, thermal noise, or cosmic rays. Without error correction, a single bit flip corrupts the entire message. We need redundancy—sending extra bits that allow the receiver to detect and correct errors.
Hamming(7,4) Code: Takes 4 data bits and adds 3 parity bits to create 7 total bits. This can correct 1 bit error. The redundancy (3/7 ≈ 43%) is a trade-off between reliability and bandwidth.

5. Putting It All Together: How Netflix Streams

The Full Pipeline
When you press play on Netflix, here's what happens mathematically:
1 Video Compression: Raw video frames are compressed using codecs like H.264 or H.265. Each frame is broken into blocks, and spatial and temporal redundancy is exploited using discrete cosine transforms (DCT) and motion estimation.
2 Entropy Encoding: The compressed bit stream is further compressed using Huffman coding or arithmetic coding to achieve near-Shannon entropy.
3 Packetisation: The stream is divided into chunks (packets), typically 1500 bytes. This makes transmission resilient: if one packet is lost, only a small piece is affected.
4 Error Correction: Each packet is wrapped in a Reed-Solomon code (a generalisation of Hamming codes). A video frame might have 100 data packets and 20 parity packets; the receiver can reconstruct the frame even if up to 20 packets are lost.
5 Transmission: Packets are sent via the Internet using UDP (fast but unreliable) or TCP (slower but guaranteed). Netflix adapts the compression ratio in real-time based on measured bandwidth.
6 Decoding: The receiver uses the error-correcting code to reconstruct packets, then decompresses the stream, and finally decodes the video frames for display.
The Mathematical Trade-off: Every design choice is a compromise. Aggressive compression (close to Shannon entropy) means less bandwidth but higher latency and quality loss. More parity bits protect against errors but waste bandwidth. Netflix uses machine learning to adapt to your network in milliseconds.
Why This All Matters
Information theory, Huffman coding, and error-correcting codes are not theoretical curiosities—they're the mathematical foundations of modern communication. Without Shannon's entropy, we wouldn't know the theoretical limits of compression. Without Huffman codes, video files would be 2–3 times larger. Without error correction, transmitted data would be corrupted constantly. Together, they enable the digital age.
