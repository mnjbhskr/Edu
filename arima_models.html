3. Model Comparison

Compare different ARMA specifications on the same data using AIC, BIC, and out-of-sample RMSE. Lower is better for all three.

AIC vs BIC: the bias-variance trade-off in model selection
AIC = −2ℓ + 2k: Penalises by twice the number of parameters. Asymptotically selects the model that minimises prediction error (Kullback-Leibler divergence). Tends to over-fit in finite samples.
BIC = −2ℓ + k · log(n): Heavier penalty that grows with sample size. Consistent: selects the true model as n → ∞ (if it is among the candidates). More conservative than AIC.
Practical guidance: Use AIC for forecasting (minimising prediction error). Use BIC for model identification (finding the “true” model). When they disagree, report both results. Out-of-sample RMSE provides a model-free check.
Where you will see this: ARIMA models are the backbone of short-term economic forecasting. Central banks use them for GDP and inflation nowcasting. The Box-Jenkins methodology (identify, estimate, diagnose) remains the standard workflow. ARIMA also serves as a benchmark against which more complex models (VARs, machine learning) are compared.

Try This

Set p=0, d=0, q=0 (white noise model). Is the residual ACF white noise? Now try p=2, d=0, q=0. Is the ACF better or worse than the true specification p=1, d=0, q=1?

ARMA(0,0) treats all autocorrelation as residual — you will see significant ACF spikes. ARMA(2,0) approximates the true ARMA(1,1) but may use more parameters than necessary. The true spec ARMA(1,1) should give the cleanest residuals and the best AIC.
