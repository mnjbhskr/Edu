The Transformer Architecture

How all the pieces fit — the complete machine that reads and writes language

The Transformer is a single elegant design that reshaped all of AI. Every token enters at the bottom, flows upward through layers of attention and computation, and emerges at the top as a prediction. The architecture is surprisingly simple — just six ideas stacked together — yet it scales to hundreds of billions of parameters and produces intelligence that surprises even its creators.

Transformer block:
  h = x + MultiHeadAttention(LayerNorm(x))
  out = h + FFN(LayerNorm(h))
where FFN(x) = W2 · GELU(W1 · x + b1) + b2

Modern transformers use pre-norm (LayerNorm before attention/FFN) rather than post-norm (after). Pre-norm improves gradient flow in deep networks — the residual path carries unmodified gradients directly through the entire depth of the model. This connects directly to the vanishing gradient problem in Backpropagation.

The feed-forward layer has more parameters than attention — it acts as the “memory”. Attention routes information; the FFN stores knowledge. In GPT-3, each FFN has 4× the hidden dimension: dmodel=12288 expands to dff=49152.
Parameter count breakdown (one transformer block, dmodel = 12,288):
Attention:  4 × d² = 4 × 12,288² ≈ 603M parameters
  (WQ, WK, WV, WO matrices)
FFN:  2 × d × 4d = 2 × 12,288 × 49,152 ≈ 1,208M parameters
  (two linear layers with 4× expansion)
LayerNorm:  4 × d ≈ 49K parameters (negligible)
FFN / Attention ratio: ~2:1

The feed-forward network has twice as many parameters as attention. It stores factual knowledge; attention routes information. Most of a transformer’s “memory” lives in the FFN layers.

GPT-4 is widely reported to use a Mixture of Experts (MoE) architecture: instead of one large FFN, it has multiple “expert” FFN sub-networks. A learned router selects which experts process each token. This allows the model to have 1.7 trillion total parameters while activating only ~280 billion per token — achieving the capacity of a dense 1.7T model at a fraction of the compute cost.

A Transformer is just six ideas stacked: embed, attend, add, normalise, transform, repeat.

2. GPT vs BERT — Decoder vs Encoder

In GPT, token 3 can see tokens 0, 1, 2, 3. In BERT, token 3 can see all tokens.
GPT (causal): Attention(Q, K, V) = softmax(QKT/√dk + M)V
  where Mij = 0 if i ≥ j, −∞ if i < j
BERT (bidirectional): same formula, but M = 0 everywhere (no masking)
GPT predicts what comes next (generation). BERT understands what is already there (comprehension). Same architecture, different masking. This single difference — a triangular matrix of negative infinities — is the line between a writer and a reader.

A causal mask turns an encoder into a decoder — the Transformer's versatility comes from this one matrix.

Why This Matters

The Transformer is the foundation of GPT, BERT, Claude, Gemini, and nearly every modern AI system. Understanding its architecture — how embedding, attention, feed-forward layers, and residual connections fit together — is understanding the engine of the AI revolution. The design has remained remarkably stable since the original 2017 paper; what changed was scale.

GPT-3 stacks 96 of these blocks with dmodel=12288. GPT-4 is believed to use a mixture-of-experts variant. But the core loop — attend, transform, add, normalise — is the same pattern you see on this page.
