Agentic AI Systems

When a language model takes action — the feedback loop that creates agency

A language model that only responds to questions is a chatbot. A language model that can observe its environment, decide what to do, take action, and learn from the result is an agent. The difference is a loop — the same observe-think-act cycle repeated until the task is done. This is the architecture behind Claude Code, AutoGPT, and every AI system that does real work in the world.

The difference between a chatbot and an agent is the loop. A chatbot responds once. An agent acts, observes consequences, and revises its plan — potentially for thousands of steps.
Cost of the loop: Each iteration requires at least one forward pass through the LLM (often multiple, if the model reasons before acting). A 10-step loop costs 10× more compute than a single response. This is why agentic systems are significantly more expensive to run — and why termination conditions matter.

Agency emerges from iteration — the same loop that lets you learn from experience lets an LLM accomplish complex tasks.

Agentic AI is the reason this technology might be transformative rather than just useful. A model that can write code, run it, observe errors, fix them, and iterate is qualitatively different from a model that only answers questions.
Failure modes: When a tool returns an unexpected format, the model may misinterpret it and spiral — each retry compounding the error. Robust agents need termination conditions: a maximum iteration count (e.g. 25 steps), a confidence threshold, or an escalation path (“I’m stuck — asking the user”). Without these, an agent can loop indefinitely, consuming compute and producing nonsense.

Planning turns a language model into a problem solver — decompose, execute, adapt.

Why This Matters

This page describes the architecture you are using right now. Claude Code is an agentic system — it observes your codebase, thinks about what to do, takes action, and iterates. Every file it reads, every edit it makes, every search it runs is one turn of the observe-think-act loop visualised above.

The ReAct pattern (Yao et al., 2022) showed that interleaving reasoning with action dramatically improves performance on complex tasks. Park et al. (2023) demonstrated that agentic architectures can produce emergent social behaviour in simulated environments. These are not theoretical constructs — they are the engineering patterns behind every modern AI assistant.
