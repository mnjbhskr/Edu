Numerical Methods

●●●● Intermediate

Finding roots when algebra fails — iterate, bisect, and converge

1. Introduction

Many equations have no neat algebraic solution. The equation x = cos(x) cannot be rearranged to isolate x. Neither can ex = 3x, or x3 − 3x − 5 = 0 (though this last one does have a closed form via the cubic formula, numerical methods are far more practical).

Numerical methods find approximate solutions to any desired accuracy by starting with a guess and improving it step by step. Each method trades off speed, reliability, and simplicity in different ways.

The foundation is the change of sign principle: if f(a) and f(b) have opposite signs, and f is continuous on [a, b], then at least one root lies between a and b. This is a consequence of the intermediate value theorem.

If f(a) × f(b) < 0 and f is continuous on [a, b], then ∃ c ∈ (a, b) such that f(c) = 0
Numerical methods are not just a fallback — they underpin modern engineering, physics, and finance. Every time your calculator solves an equation, it is running an algorithm much like the ones on this page.

2. Bisection Method

Interactive — Bisection Visualiser

The bisection method repeatedly halves the interval containing a root. At each step, we find the midpoint c = (a + b)/2, evaluate f(c), and replace whichever endpoint shares the same sign. Convergence is guaranteed but slow — each step gains roughly one decimal digit of accuracy every 3.3 iterations.

Algorithm:
1. Compute c = (a + b) / 2
2. If f(a) × f(c) < 0, set b = c
3. Otherwise, set a = c
4. Repeat until |b − a| < tolerance
The bisection method gains exactly one bit of precision per step. After n iterations the interval width is (b − a)/2n. For 10 decimal places, you need about 34 steps — slow but absolutely reliable.

3. Fixed-Point Iteration

Interactive — Cobweb Diagram

Rearrange f(x) = 0 into the form x = g(x), then iterate: xn+1 = g(xn). The sequence converges to a fixed point where the curve y = g(x) crosses the line y = x — provided |g′(x)| < 1 near the root.

Convergence condition: |g′(x*)| < 1 at the fixed point x*
If |g′(x*)| > 1, the iteration diverges
The cobweb diagram makes convergence visible. When |g′| < 1 the “steps” spiral inward to the fixed point. When |g′| > 1 they spiral outward — the iteration diverges. Try switching between convergent and divergent rearrangements to see the difference.

4. Newton-Raphson Method

Interactive — Tangent-Line Visualiser

The Newton-Raphson method is the workhorse of numerical root-finding. At each step, draw the tangent to f(x) at the current estimate and follow it to the x-axis. Where the tangent crosses zero becomes the next estimate.

xn+1 = xn − f(xn) / f′(xn)

Convergence is quadratic when it works: the number of correct decimal places roughly doubles with each step. But it can fail if f′(xn) is near zero, or if the starting point is too far from the root.

Newton-Raphson’s speed comes at a cost: you need the derivative f′(x). For complicated functions this can be expensive, which motivates the secant method (replacing f′ with a finite-difference approximation) — a topic for further study.

5. Comparing Methods

Method Summary

MethodSpeedReliabilityRequirements
Bisection Slow (linear) Guaranteed Sign change on [a, b]
Fixed-point iteration Medium (linear) Conditional |g′(x*)| < 1
Newton-Raphson Fast (quadratic) Conditional f′(x) available, good start

Error bounds: After n bisection steps, the error is at most (b − a)/2n. For Newton-Raphson, the error satisfies |en+1| ≈ C · |en|² where C depends on f″ near the root.

▶ When Newton-Raphson fails

Newton-Raphson can fail in several ways:

1. Division by zero. If f′(xn) = 0, the tangent is horizontal and never crosses the x-axis. The formula produces ±∞.

2. Cycling. For some functions (e.g. f(x) = x3 − 2x + 2 starting from x0 = 0), the iterates oscillate between two values and never converge.

3. Overshoot. If the function has a steep slope followed by a gentle slope, the tangent may “overshoot” far from the root, sending the iterate to a distant region of the function.

Remedy: In practice, Newton-Raphson is often combined with bisection as a “safety net”: accept the Newton step if it stays within the bracketed interval, otherwise fall back to a bisection step. This gives quadratic speed with guaranteed convergence.

6. Worked Examples

Example 1 — Newton-Raphson to 4 d.p.

Show that x3 − 3x − 5 = 0 has a root between 2 and 3, then use Newton-Raphson to find it correct to 4 decimal places.

Change of sign:
  f(2) = 8 − 6 − 5 = −3 < 0
  f(3) = 27 − 9 − 5 = +13 > 0
Sign change ⇒ root in (2, 3) ✓
Newton-Raphson: f(x) = x³ − 3x − 5,   f′(x) = 3x² − 3
n=0: x0 = 2
n=1: x1 = 2 − (−3)/(9) = 2 + 0.3333… = 2.333333
n=2: x2 = 2.333333 − f(2.333333)/f′(2.333333) = 2.280970
n=3: x3 = 2.279020
n=4: x4 = 2.279019
Root: x ≈ 2.2790 (4 d.p.)

Example 2 — Fixed-Point Iteration

Use the iteration xn+1 = √(3 + 5/xn) with x0 = 2 to find a root of x3 − 3x − 5 = 0.

Rearrangement: x³ = 3x + 5 ⇒ x² = 3 + 5/x ⇒ x = √(3 + 5/x)
n=0: x0 = 2.000000
n=1: x1 = √(3 + 5/2) = √5.5 = 2.345208
n=2: x2 = √(3 + 5/2.345208) = 2.274930
n=3: x3 = 2.279783
n=4: x4 = 2.278963
n=5: x5 = 2.279101
n=6: x6 = 2.279078
Converging to: x ≈ 2.2790 (same root as Example 1, but slower convergence)
Notice how Newton-Raphson reached 4 d.p. accuracy in 4 steps, while fixed-point iteration needed about 6. This illustrates the difference between quadratic and linear convergence. Both methods find the same root — but at very different speeds.
