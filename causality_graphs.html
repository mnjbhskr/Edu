Cause, Effect & Structure

Correlation is symmetric. Causation is not. The difference is a mathematical structure called a directed acyclic graph

Core idea Correlation is symmetric; causation is not — and a directed acyclic graph captures the difference, subject to the assumptions it encodes.
Critical caveat — DAGs encode assumptions, not observations
A causal DAG is not discovered from data alone. It encodes the researcher's assumptions about the causal structure of the world — which variables affect which others, and in which direction. These assumptions cannot be verified from observational data without additional conditions (identifiability). A correctly specified DAG enables valid causal inference; an incorrectly specified DAG produces confidently wrong conclusions. When you see a causal claim supported by a DAG, the right question is not 'is the statistical analysis correct?' but 'are the DAG's structural assumptions justified?' This is where domain knowledge is irreplaceable.

1. Simpson's Paradox — When Aggregation Lies

MEDICAL TREATMENT — AGGREGATED vs STRATIFIED

0.70 
Simpson's Paradox:

P(Recovery | Treatment) > P(Recovery | No Treatment)   [in aggregate]

but:

P(Recovery | Treatment, Mild) < P(Recovery | No Treatment, Mild)
P(Recovery | Treatment, Severe) < P(Recovery | No Treatment, Severe)

Causal DAG:
    Severity → Treatment
    Severity → Recovery
    Treatment → Recovery    ← the effect we want

Without controlling for Severity, we measure the wrong quantity.
💡 Simpson's Paradox arises regularly in real data — in clinical trials, education statistics, hiring data. It is not a curiosity but a fundamental warning that aggregate associations can reverse when stratified. The resolution requires knowing the causal structure — and that knowledge is external to the data. It lives in the DAG.

2. The Do-Calculus — Observation vs Intervention

CAUSAL DAG BUILDER

Treatment X: Outcome Y:
Click a source node, then a target node, to draw a directed arrow. Toggle remove-mode to delete.
Confounder Mediator Collider X (treatment) Y (outcome) 
Observational query:   P(Y | X=x)
    "Probability of Y in the subpopulation where X=x."
    Affected by confounding.

Interventional query:  P(Y | do(X=x))
    "Probability of Y if we force X to equal x."
    Removes all incoming arrows to X.

These are equal ONLY when there is no confounding.

Randomised controlled trials exploit this:
    Random assignment ensures do(X=x) = observe X=x
    because assignment is independent of all confounders.
💡 The fundamental challenge of observational data is that we observe correlations but want interventions. A DAG makes the distinction precise: P(Y|X=x) asks about a subpopulation; P(Y|do(X=x)) asks about an intervention. These are different mathematical objects, and confusing them is the source of most erroneous causal claims in data analysis.

3. Bayesian Networks — Belief Propagation

MEDICAL DIAGNOSIS NETWORK

Click a node to cycle: unobserved → observed TRUE → observed FALSE → unobserved. Posterior beliefs recompute via exact enumeration.
💡 A Bayesian network is a probabilistic DAG: each node carries a conditional probability table given its parents, and the joint distribution factorises accordingly. Observing one node causes beliefs about unobserved nodes to update — forwards along causal arrows (causes explain effects) and backwards (effects update beliefs about causes). This bidirectional reasoning is what makes probabilistic graphical models so powerful in diagnosis.
What have we learnt?
Correlation is symmetric: if A correlates with B, then B correlates with A. Causation is asymmetric: if A causes B, it does not follow that B causes A. A directed acyclic graph encodes this asymmetry mathematically — the direction of an edge is a statement about mechanism, not just association. Simpson's Paradox shows that this distinction is not philosophical but practical: the same data can support opposite conclusions depending on whether you account for causal structure. The do-operator separates observation from intervention, and Bayesian networks turn a DAG into a probabilistic reasoning engine. The choice of representation — causal graph versus correlation matrix — determines what questions you can answer. The choice of representation is not neutral — it is a mathematical act with consequences. The power of causal graphs comes with a responsibility: the graph is a model of assumptions, not a model of truth. The mathematics is only as reliable as the causal structure it encodes.
