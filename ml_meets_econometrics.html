2. Bias-Variance Tradeoff

As model complexity increases, in-sample error keeps falling. But out-of-sample error follows a U-shape: too simple means bias, too complex means variance. The optimal complexity minimises test error.

3. Double/Debiased ML

The partial-out procedure: use ML to predict treatment D from controls X, and outcome Y from controls X. Regress residuals on residuals. The result is root-n consistent for the causal parameter — even when the nuisance functions are estimated with ML.

Key insight: Machine learning does not solve the identification problem — it solves the nuisance parameter estimation problem. You still need a credible research design; ML simply handles the high-dimensional controls more efficiently.
