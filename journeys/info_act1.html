The Language of Information · Act I
What Is Information?
📡

You receive a message: "The sun rose this morning."

How much information did that contain?

Almost none. You already knew it would happen.

Now consider: "The coin came up heads ten times in a row."

That's surprising. That's informative.

Claude Shannon, 1948

In a landmark paper, a 32-year-old Bell Labs mathematician gave "information" a precise mathematical definition. Not based on meaning, not based on usefulness — based purely on surprise. The rarer an event, the more information its occurrence carries. This single idea founded the digital age.

Comparing messages
More surprising = more informative

Tap a message from each pair. Which one tells you more? Which one surprises you more?

Try This

In each pair, tap the message that surprises you more. After tapping, the information content in bits appears. The more surprising message always carries more bits. This is not a coincidence — it is the definition.

💡 Shannon's key insight

Information content is I(p) = log₂(1/p) = −log₂(p) bits. An event with probability 1 carries 0 bits — you already knew. An event with probability ½ carries 1 bit. An event with probability 1/1024 carries 10 bits. Surprise and information are the same thing.

What To Notice

A fair coin flip (p = 0.5) carries exactly 1 bit. This is where the unit "bit" gets its name — one binary digit, one yes/no question. A die roll (p = 1/6) carries about 2.58 bits. The number of bits is exactly the number of yes/no questions needed to identify the outcome.

💡 The zero and the infinite

A certain event (p=1) carries exactly 0 bits — you learn nothing new. An impossible event (p=0) would carry infinite information — but it can never occur, so you never receive that infinite surprise. Shannon's definition is perfectly calibrated at both ends.

💡 What you learned

Information content I(p) = log₂(1/p) bits. This is not a metaphor — it is a precise quantity. A fair coin flip carries exactly 1 bit. The log₂ means: "how many yes/no questions would you need to determine this outcome?" Information is the number of yes/no questions answered.

Why It Matters

Shannon's definition of information transformed communication engineering into an exact science. Before 1948, engineers used trial and error. After Shannon, they knew the theoretical limits — precisely how much data any channel can carry, and how efficiently any source can be compressed. Every phone call, every WiFi signal, every streaming video operates within Shannon's framework.

Coming in Act II — Entropy

If a source sends many different symbols with different probabilities, what is the average information per symbol? Shannon's entropy H measures exactly this — and it turns out to be the fundamental limit on how efficiently any message can be compressed.
