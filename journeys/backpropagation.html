Back­propagation

The chain rule at scale — how gradients flow backward through billions of operations to teach a neural network what to change.

The Problem of Credit Assignment

A neural network makes a prediction. The prediction is wrong. Which parameters caused the error — and by how much? This is the credit assignment problem, and it is one of the oldest challenges in machine learning.

With millions of parameters all contributing to an output, how do you know which ones to adjust, and in which direction? The answer is backpropagation — the systematic application of the chain rule of calculus to compute every gradient simultaneously in a single backward pass through the network.

History — Rediscovered Three Times

Backpropagation was independently discovered by Seppo Linnainmaa (1970), Paul Werbos (1974), and Rumelhart, Hinton & Williams (1986). The 1986 paper, Learning representations by back-propagating errors, demonstrated it working on practical problems and launched the first neural network renaissance. The mathematics had waited twelve years for the hardware to catch up.

The Intuition: Wiggle and Observe

Before diving into the chain rule, consider a simpler idea. You have a weight in your network. You want to know: if I change this weight by a tiny amount, how does the error change?

You could literally try it — nudge the weight up by 0.001, recompute the output, measure the new error. The difference tells you the gradient for that one weight. But with millions of weights, this brute-force approach is far too slow. Backpropagation computes all gradients at once, in a single backward sweep.

The Core Idea — Simplified

Change a weight slightly. Observe the effect on the error. If the error goes up, the weight should decrease. If the error goes down, the weight should increase. The gradient tells you both the direction and the magnitude. Backpropagation is the efficient algorithm for computing this gradient for every weight simultaneously.

The Chain Rule — Foundation of Everything

If f(g(x)) is a composition of two functions, the derivative of the composition is:

Chain rule: d/dx [f(g(x))] = f'(g(x)) * g'(x) In other words: the derivative of a composed function is the product of the derivatives of each component, evaluated at the right place. For a chain of 3 functions: L = f(g(h(x))) dL/dx = dL/df * df/dg * dg/dh * dh/dx Neural networks are just very long chains. Backpropagation is just the chain rule applied systematically, layer by layer, from output to input.

What To Notice
  • Error flows backward: from output to hidden layer to input. This is why it is called "back"-propagation.
  • Arrow brightness and thickness indicate gradient magnitude — some weights matter more than others.
  • Weights with large gradients (bright red arrows) will change the most. Weights with small gradients barely move.
  • After the update, the network's prediction moves closer to the target.
Why It Matters
  • Backpropagation makes learning possible. Without it, we would need to test each weight individually — impossibly slow for millions of parameters.
  • It computes all gradients in one pass, making it only ~2x the cost of a forward pass.
  • Every modern AI system — GPT, DALL-E, AlphaFold — learns through backpropagation.

The Computation Graph

Any computation can be represented as a directed acyclic graph (DAG) where nodes are operations and edges are the flow of values. Forward propagation traverses this graph from inputs to outputs. Backpropagation traverses it in reverse, computing gradients at each node using the chain rule.

Simplified View of the Algorithm

The intuition (no matrices needed): 1. FORWARD: Send input through the network, computing output at each layer. Store all intermediate values (you will need them). 2. COMPUTE ERROR: Compare output to target. How wrong are we? 3. BACKWARD: For each layer, from last to first: - "How much did THIS layer contribute to the error?" - Answer: multiply incoming error by local derivative - Pass the result to the previous layer 4. UPDATE: For each weight: weight = weight - learning_rate * gradient (Nudge each weight in the direction that reduces error)

Why the Forward Pass Must Store Everything

Notice that the backward pass uses intermediate values computed during the forward pass. This is why training a large model requires much more memory than running inference. During training, you must keep every intermediate activation in memory until its gradient has been computed.

For GPT-3 with 96 layers, this requires storing activations for every layer, for every token in the batch. At a batch size of 1 million tokens, this amounts to hundreds of gigabytes of activations — before even storing the parameters.

Gradient checkpointing

To reduce memory, gradient checkpointing recomputes intermediate activations during the backward pass rather than storing them. This trades compute for memory: the forward pass is run twice, but activation memory drops from O(depth) to O(sqrt(depth)). Used in training the largest models.

Vanishing and Exploding Gradients

The chain rule multiplies many derivatives together. If each derivative is less than 1 (as with the sigmoid activation, which saturates), the product shrinks exponentially with depth. Vanishing gradients mean early layers receive almost no learning signal — the gradient that reaches layer 1 of a 100-layer network may be 10^-30 times the gradient at layer 100.

Conversely, if derivatives are large, the product explodes. Exploding gradients cause parameter updates so large the network destabilises. The solution: gradient clipping (cap the gradient norm) and careful architecture choices (residual connections, normalisation layers).

Sigmoid derivative: sigmoid'(x) = sigmoid(x)(1-sigmoid(x)) <= 0.25 For a 10-layer network with sigmoid activations: gradient at layer 0 ~ 0.25^10 ~ 9.5 x 10^-7 -- essentially zero ReLU derivative: max(0,x)' = 0 if x<0, else 1 Gradients pass through unchanged (if x>0) -> no vanishing But "dead ReLU": if x<0, gradient = 0, neuron stops learning Solution: Residual connections (ResNet, 2015): y = F(x) + x -- identity shortcut dy/dx = dF/dx + 1 -- gradient always has +1, never vanishes
Why modern networks are deep but trainable

Residual connections (He et al., 2015) solved the vanishing gradient problem for very deep networks. By adding an identity shortcut around each block, gradients always have a direct path to early layers. ResNet-152 (152 layers) trained successfully where earlier 20-layer networks had failed. Every modern large model uses residual connections.
