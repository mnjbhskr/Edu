The Attention Mechanism

How a model learns which parts of its input to focus on — the mathematical operation that made transformers possible.

The Sequence Problem

Before attention, neural networks processed sequences word by word, passing a fixed-size "memory" (hidden state) from one word to the next. By the time the network reached the end of a long sentence, early words had been compressed and distorted by all subsequent processing. Long-range dependencies were lost.

The attention mechanism solves this by allowing every position in a sequence to directly attend to every other position — in a single operation. The cost of relating position 1 to position 500 is the same as relating position 1 to position 2. Distance no longer degrades information.

Vaswani et al., "Attention Is All You Need" — 2017

The paper that introduced the transformer architecture replaced recurrence entirely with attention. Its title was deliberately provocative — and correct. The transformer's 65,000 citations make it one of the most influential papers in the history of machine learning. Every large language model — GPT, Claude, Gemini, LLaMA — is built on this idea.

Pronoun Resolution: Why Attention Matters

Consider this sentence:

"The animal didn't cross the street because it was too tired."

Which word does "it" refer to? A human reader instantly knows: "it" refers to "the animal" — the animal was too tired to cross. But how does a model figure this out?

The attention mechanism lets the model, when processing the word "it", look back at every other word in the sentence and compute a relevance score. It learns to assign high attention weight to "animal" because the semantic context ("tired" is a property of animals, not streets) makes that connection strong.

What To Notice

  • The model can attend to distant words just as easily as nearby ones — this is the key advantage over older recurrent networks.
  • Different attention heads (in multi-head attention) can resolve different types of relationships simultaneously.
Why It Matters
  • Pronoun resolution is one of the hardest problems in NLP. Attention solves it elegantly.
  • This same mechanism handles all long-range dependencies: subject-verb agreement, coreference, logical reasoning.
  • Without attention, models would need to compress all context into a fixed-size vector — losing information.

Query, Key, Value

Attention is modelled on a database lookup — but a soft, differentiable one. Every token in the sequence plays three roles simultaneously:

Q
Query
"What am I looking for?" A vector representing what this position wants to find in other positions.
K
Key
"What do I contain?" A vector representing what information this position has to offer.
V
Value
"What do I actually send?" A vector containing the information to be passed if this position is attended to.

The attention weight between position i and position j is computed by matching query i against key j: a high dot product means "position i finds position j relevant." The weights are then normalised with softmax and used to take a weighted average of all values.

Scaled dot-product attention: Attention(Q, K, V) = softmax( Q*K^T / sqrt(d_k) ) * V Q in R^(n x d_k) -- n queries, each d_k-dimensional K in R^(n x d_k) -- n keys V in R^(n x d_v) -- n values Q*K^T in R^(n x n) -- attention score matrix (all pairs) /sqrt(d_k) -- scale to prevent softmax saturation softmax(.) -- normalise each row to sum to 1 * V -- weighted sum of values Output at position i = Sum_j softmax(Q_i*K_j/sqrt(d_k)) * V_j = weighted average of all values, where weights reflect query-key similarity

Why divide by sqrt(d_k)?

Without the scaling factor, dot products between d_k-dimensional vectors grow proportionally to d_k. With typical values of d_k = 64, raw dot products can reach magnitudes of +/-8, which push the softmax into saturation regions where gradients vanish. Dividing by sqrt(64) = 8 keeps the inputs to softmax in a well-behaved range.

Why It Matters
  • The attention matrix is the model's "reasoning trace" — it shows what the model considers relevant.
  • Interpretability researchers study attention patterns to understand what models have learned.
  • The O(n^2) cost of this matrix is the fundamental bottleneck of transformer models for long sequences.

Multi-Head Attention

A single attention head can only capture one type of relationship at a time. Multi-head attention runs h parallel attention heads, each with different learned projections for Q, K, V. The outputs are concatenated and projected again. This allows the model to simultaneously attend to different types of relationships — syntactic structure, semantic similarity, coreference — from different representation subspaces.

Multi-head attention: MultiHead(Q, K, V) = Concat(head_1, ..., head_h) * W^O where head_i = Attention(Q*W_i^Q, K*W_i^K, V*W_i^V) Each head has its own learned projections: W_i^Q, W_i^K, W_i^V h = 8 to 96 heads depending on model size d_k = d_model / h (head dimension shrinks to keep total cost constant) GPT-3: 96 attention heads per layer x 96 layers = 9,216 total heads Each head models a different kind of relationship in the sequence
What do attention heads learn?

Interpretability researchers have found that individual attention heads specialise. Some learn to track coreference (matching pronouns to nouns). Some learn syntactic subject-verb agreement. Some track positional patterns. Some seem to detect named entities. The model discovers these structured relationships without being told to look for them — they emerge from the task of next-token prediction.

Computational Complexity

The attention matrix Q*K^T has n^2 entries, where n is the sequence length. This means attention scales as O(n^2) in both time and memory — a significant limitation for very long sequences. For n = 4,096 tokens, the attention matrix has 16 million entries per head per layer.

This quadratic scaling has motivated a large research effort into sparse and linear attention approximations (Longformer, Flash Attention, Ring Attention), which reduce complexity to O(n log n) or O(n) while preserving most of the expressiveness.

Flash Attention — making the quadratic tractable

Flash Attention (Dao et al., 2022) does not change the mathematical result of attention — it computes exactly the same thing. Instead, it reorganises the computation to be memory-aware, processing attention in tiles that fit in fast GPU SRAM rather than slow HBM memory. The result: 2-4x speedup and 10-20x memory reduction. Most modern training now uses Flash Attention.
