The Language of Information · Act II
Entropy: the average surprise
🎲

A single event has information content I(p) = −log₂(p). But what if a source sends many different symbols with different probabilities?

What is the average information per symbol? The answer is Shannon entropy — and it is one of the most important numbers in all of science.

Claude Shannon, 1948

Shannon defined entropy H = −Σ pᵢ log₂ pᵢ. He was about to call it "uncertainty" but John von Neumann told him: "Call it entropy. Nobody knows what entropy really is, so in a debate you'll always have the advantage." The name stuck. It turned out to be the same entropy as in thermodynamics — not by coincidence.

💡 Maximum uncertainty at p = 0.5

The entropy hill peaks at p = 0.5 (H = 1 bit) — when the coin is fair, you are most uncertain about the outcome. It reaches zero at p = 0 and p = 1 — when the outcome is certain, there is nothing to learn. Entropy measures uncertainty, not randomness.

💡 Entropy of a fair die

For a fair 6-sided die: H = log₂(6) ≈ 2.58 bits. For a fair 52-card deck: H = log₂(52) ≈ 5.70 bits. For n equally-likely outcomes: H = log₂(n) bits. Maximum entropy = maximum uncertainty = all outcomes equally likely.

Why It Matters

Entropy is the fundamental limit on data compression. You cannot compress data below its entropy — no algorithm, however clever, can beat this limit. It also measures uncertainty: high entropy = high uncertainty = hard to predict. This connects directly to AI: a language model's training loss is cross-entropy, measuring how surprised the model is by the actual next token.

The Language of Information · Act III
Compression: the entropy limit
🗜️

Why can you compress a text file to 30% of its size, but a ZIP file barely shrinks at all?

Shannon's source coding theorem gives the answer: you cannot compress data below its entropy. No algorithm — however clever — can beat this limit.

Shannon's First Theorem (1948)

A source producing symbols with entropy H bits/symbol requires at least H bits/symbol on average to encode. English text has entropy ≈ 1.0–1.3 bits/character (taking into account word-level patterns) — which is why it compresses so well. Random data has entropy ≈ 8 bits/byte and cannot be compressed at all.

Huffman coding
Frequent symbols get shorter codes

Type any text below. The encoder will count character frequencies and build an optimal code — shorter codes for common characters, longer for rare ones.

Try This

Type "AAAAAAA" (all the same letter) — entropy drops to 0, the text is maximally compressible. Then type random characters — entropy rises, compression becomes impossible. Try "information theory" to see Huffman coding in action on real text.

What To Notice

Frequent characters get short codes (fewer bits), rare characters get long codes (more bits). The average code length is always close to — but never below — the entropy H. This is Shannon's source coding theorem in action: entropy is the absolute minimum description length.

💡 The compression limit

The average Huffman code length approaches — but never goes below — the entropy H of the source. This is Shannon's theorem made visible. Try typing all the same letter (entropy = 0, tiny file). Try random keys (entropy ≈ 8 bits/char, incompressible).

The Language of Information · Act IV
Noisy channels: the impossible made possible
📻

You want to send a message through a channel that randomly flips bits. Can you communicate reliably?

Intuition says: if the channel is noisy enough, no. Shannon's theorem says: you can always communicate reliably, as long as your rate is below the channel capacity. This was the most shocking result in the paper.

Shannon's Second Theorem (1948)

Channel capacity: C = B log₂(1 + S/N) bits/second, where B is bandwidth and S/N is the signal-to-noise ratio. For any transmission rate below C, there exist error-correcting codes that make the error probability arbitrarily small. Above C: impossible. The capacity is an absolute limit — and an absolute guarantee.

The noisy channel

Set the error probability, then send the message. Watch bits flip. Then add redundancy — and watch the errors disappear.

What To Notice

With enough redundancy, even a very noisy channel can transmit data reliably. Shannon proved this is always possible below channel capacity C. Above C, reliable communication is impossible no matter what you do. This was the most surprising result of the entire paper.

💡 Why redundancy works

With 3× repetition: each bit sent as "000" or "111". A single flip gives "001" — majority vote recovers the original. Hamming codes do this more efficiently. Deep-space probes use sophisticated codes that correct many errors with minimal overhead. Redundancy turns a noisy channel into a reliable one.

💡 Error correction in the real world

QR codes carry 30% redundancy — scan a partially damaged code and it still works. CDs use Reed-Solomon codes that can recover from a 4000-bit burst error. The Voyager probe (now in interstellar space) uses a Golay code. 5G uses turbo codes approaching Shannon capacity to within a fraction of a percent.

The Language of Information · Act V
Information is physical
🧬

Shannon's theory started as an engineering framework for telegraph cables. But the mathematics reached far beyond communication — into biology, thermodynamics, and the deepest questions about the nature of reality.

Information is not abstract. It is physical.

Why It Matters

Shannon's information theory connects to three pillars of modern technology: Data compression (ZIP, JPEG, MP3 all approach the entropy limit). Communication (5G, WiFi, satellite links all operate near channel capacity). Artificial intelligence (language models minimise cross-entropy, which is Shannon entropy applied to predictions). One 1948 paper — three revolutions.
