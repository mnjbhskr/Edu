The Broken Machine · Act IV
The Machine That Cannot Decide
⚙

Alan Turing. Cambridge, 1936. Age 24.

He had read Gödel. He understood, perhaps better than anyone, what the theorems meant.

He asked a different question. Not “what can’t mathematics prove?” but “what can’t a machine compute?”

To answer it, he had to invent, precisely and for the first time, what a machine is.

Alan Turing, 1912–1954

A runner, a mathematician, a codebreaker. Turing’s 1936 paper “On Computable Numbers” defined computation itself — and proved its limits — before any programmable computer existed. He was 24 years old.

Try This

Before continuing: can you imagine a program that analyses other programs and always correctly decides whether they will finish running or loop forever? Hold that thought.

What To Notice

This act echoes Acts II and III. The same diagonal argument appears a third time. Watch for it. The déjà vu is deliberate.

Why It Matters

Turing’s proof did two things at once: it showed that certain problems are impossible to solve by computation — and it gave us the precise definition of what a computer is. The limit and the invention arrived together.

Discovery
The Turing Machine

A Turing machine is absurdly simple. An infinite tape of cells. A read/write head that moves one cell at a time. A state register — the machine’s current mood. And a rule table: if in state X reading symbol Y, write Z, move left or right, enter new state.

That is all a computer is.

Turing asked: is there a single universal algorithm that can examine ANY program and its input, and decide: “this will halt” or “this will run forever”?

Can the machine be its own judge?

The Lesson

Programs 4, 5, and the bonus are not trick questions. They are equivalent to open problems in mathematics. If you could decide halting, you could resolve the greatest unsolved conjectures by simply running the relevant program and checking whether it stops.

This is not a party trick. The Halting Problem is genuinely hard — not because computers are slow, but because halting is connected to mathematical truth itself.

Discovery
The Diagonal Returns

Here it is again. The move Cantor used to prove some infinities are larger than others. The move Gödel used to construct his unprovable statement.

Turing is about to use it too. Watch carefully — you’ve seen this before.

Step 1 — Suppose a Halt-Checker Exists
Assume — for contradiction — that there IS a program H that can decide, for any program P and input I, whether P(I) halts.
H(P, I) = HALTS   if P terminates on input I
H(P, I) = LOOPS   if P runs forever on input I
Step 2 — Build a New Program D
Using H, build a new program D that takes any program P as input:
D(P):
  Run H(P, P)
  If H says “halts” → D loops forever
  If H says “loops” → D halts immediately
D does the opposite of what H predicts. A legal program, built from a legal component.
Step 3 — What Does D(D) Do?
If D(D) Halts…
1. D(D) halts
2. H was asked “does D(D) halt?” and answered “halts”
3. But D’s rule says: if H answers “halts” → loop forever
4. So D(D) should loop — but we said it halts
⚠ CONTRADICTION
If D(D) Loops…
1. D(D) loops forever
2. H was asked “does D(D) halt?” and answered “loops”
3. But D’s rule says: if H answers “loops” → halt immediately
4. So D(D) should halt — but we said it loops
⚠ CONTRADICTION

Both branches lead to contradiction. The contradiction came from assuming H exists.

H cannot exist. No program can decide halting for all programs.

This is not a limitation of current technology. It is a theorem. No program — on any computer, past or future, however fast — can solve the Halting Problem.

The Diagonal Echo

You have seen this argument before.

Cantor assumed a list of all real numbers — and constructed a number not on the list, by changing each diagonal digit.

Gödel assumed a complete formal system — and constructed a statement that says “I am not provable.”

Turing assumed a halt-checking program — and built a program that contradicts itself when run on itself.

One argument. Three revelations. Self-reference combined with negation always breaks the assumption of completeness.

Proof
Undecidability
Intuitive
Precise

A program that checks whether programs halt would need to simulate what those programs do. But some programs run forever — so the checker would have to run forever simulating them, which means the checker itself never gives an answer.

But more than this: if you could check halting, you could resolve the great open conjectures of mathematics by building programs that search for counterexamples.

The undecidability of halting is, in a precise sense, the reason mathematics still has unsolved problems.

Theorem (Turing, 1936). The Halting Problem is undecidable: there is no Turing machine H such that for all machines P and inputs I,

H(P, I) terminates and correctly outputs “halts” or “loops”.

Proof. By diagonal contradiction, as above. Assume H exists; construct D that does the opposite of H’s prediction when given itself; D(D) leads to contradiction on both branches; therefore H does not exist. □

Corollary (Rice’s Theorem). The class of undecidable problems is vast. Any non-trivial semantic property of programs — “does this program produce output X?”, “does this program ever enter state S?” — is undecidable.

The Shape of The Argument

Cantor 1891: Assume a list → construct what’s not on it → contradiction.

Gödel 1931: Assume completeness → construct what can’t be proved → contradiction.

Turing 1936: Assume decidability → construct what contradicts the decision → contradiction.

The diagonal argument is the mathematics of impossibility.

Connection
The Computer Is Born

Turing published his proof in 1936.

To prove something was undecidable, he had to precisely define what it would mean for something to be decidable in the first place.

That precise definition — the Turing Machine — was the first rigorous mathematical model of computation.

Every computer ever built — every phone, laptop, supercomputer, quantum computer — is, in terms of what it can and cannot compute, equivalent to a Turing Machine. They are faster. They have more memory. They cannot solve different problems.

The Paradox

The computer was invented to prove what computers cannot do.

Undecidable Problems — Proven Impossibilities
Virus Detection
Whether an arbitrary program contains a virus is undecidable. This is why antivirus software is fundamentally limited — it cannot be perfect.
Program Equivalence
Whether two arbitrary programs compute the same function is undecidable. No tool can ever guarantee that a rewrite of your code does exactly what the original did.
Output Prediction
Whether an arbitrary program will ever output a specific string is undecidable. You cannot, in general, predict what a program will produce without running it.
Mathematical Provability
Whether a mathematical theorem is provable from given axioms is undecidable. Turing’s result and Gödel’s meet here: the limits of computation and the limits of proof are the same limit.

These are not unsolved engineering problems. They are proven mathematical impossibilities.

The walls are real. And knowing exactly where they are is one of the greatest achievements of twentieth-century thought.
