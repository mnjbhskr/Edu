Chain-of-Thought Reasoning

Thinking out loud — how scratchpad reasoning unlocks harder problems

Large language models predict the next token. For easy questions, a single forward pass is enough. But for harder problems — arithmetic, logic, planning — the model needs to think step by step. Chain-of-thought prompting gives the model a scratchpad: each reasoning token becomes an intermediate computation that shapes the final answer. The result is striking — the same model, with the same weights, solves problems it previously got wrong.

Without chain-of-thought, the model uses a single forward pass to answer. With CoT, it uses hundreds of forward passes — one per reasoning token. Each token is a small computation that feeds into the next. More compute = better answers.
When CoT hurts — overthinking a trivial problem
Problem: “What is 2 + 3?”
Direct: P(“5”) = 0.99 → Correct ✓
CoT: “Let me think… 2 is even, 3 is odd, even + odd = odd, the smallest odd number greater than 2+3… wait, 2+3 = 5” → Correct but wasted 10× compute
For trivial problems, CoT wastes compute without improving accuracy. The benefit emerges only when the problem exceeds the model’s single-pass reasoning capacity.

Chain-of-thought reasoning works reliably only in sufficiently large models (roughly >100B parameters). Smaller models may generate plausible-looking reasoning steps that lead to wrong answers — the appearance of thought without the substance.

A single forward pass can only do so much. Chain-of-thought turns token generation into multi-step computation.

The insight behind “thinking” models (o1, o3, DeepSeek R1): spend more tokens computing before giving an answer. Intelligence can be rented with compute.

The reasoning tree turns a single guess into a systematic search — more branches explored means more problems solved.

Why This Matters

Chain-of-thought reasoning is not a trick — it is a fundamental insight about computation. A transformer has fixed depth: each forward pass does a bounded amount of serial computation. By generating intermediate tokens, the model effectively deepens its computation graph at inference time. Each reasoning token is a new layer of thought, allowing the model to solve problems that exceed the capacity of a single forward pass.

Wei et al. (2022) showed that chain-of-thought prompting enables large language models to perform complex reasoning. Yao et al. (2023) extended this with Tree of Thoughts, allowing deliberate exploration of reasoning paths. Today’s frontier models (o1, o3, DeepSeek R1) bake this into training — learning when and how to think before answering.

Emergence caveat: CoT is an emergent capability — it appears reliably only in models above roughly 100 billion parameters. Smaller models can mimic the form of step-by-step reasoning (generating plausible-looking intermediate tokens) while arriving at wrong answers. The reasoning steps look right but carry no real computation. This is a critical distinction: the appearance of thought is not the same as thought itself.
