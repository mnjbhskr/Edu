Critical Path Analysis

●●●● Intermediate

Planning projects with networks, finding the critical path, and understanding float

1. Introduction

Every project — building a house, launching a product, organising an event — consists of activities that take time and depend on one another. Some tasks cannot begin until others are complete: you cannot plaster walls before you have built them, and you cannot paint before the plaster has dried.

Critical Path Analysis (CPA) models a project as a network of activities, each with a duration and a set of dependencies. By analysing this network, we can answer the most important question in project management: What is the shortest possible time to complete the entire project?

The technique uses an activity-on-node (AoN) network. Each node represents an activity, showing its name, duration, earliest start time (EST), and latest finish time (LFT). Arrows between nodes show dependencies — which activities must be completed before others can begin.

The critical path is the longest route through the network. Activities on this path have zero float — any delay to them delays the entire project. Understanding the critical path tells managers where to focus resources and where there is flexibility.

CPA was developed independently in the late 1950s by the DuPont Corporation and the US Navy’s Polaris missile programme. It remains one of the most widely used techniques in project management, from construction to software development.

2. Building the Network

Activity-on-Node Diagrams

In an activity-on-node diagram, each activity is drawn as a box (node) containing key information. Arrows connect nodes to show precedence relationships — activity B depends on activity A if there is an arrow from A to B.

Each node typically displays:

┌──────────────────────┐
│  EST               LFT  │
│       Activity Name      │
│        Duration: d        │
│         Float: f         │
└──────────────────────┘

Worked Example: Building a House

Consider a simplified house-building project with eight activities:

ActivityDescriptionDuration (days)Depends on
A Lay foundations 5 —
B Build walls 8 A
C Install roof 6 B
D Plumbing 4 B
E Electrical wiring 3 B
F Plastering 4 C, D, E
G Painting 3 F
H Landscaping 5 A
Notice that some activities can happen in parallel. Plumbing (D) and electrical wiring (E) can both proceed at the same time once the walls are built. Landscaping (H) can happen independently of the interior work. The network captures all of these relationships.

3. Forward & Backward Pass

Computing EST and LFT

The forward pass calculates the earliest start time (EST) for each activity. We work left to right through the network:

EST(activity) = max{EST(predecessor) + duration(predecessor)} for all predecessors

Starting activities (those with no predecessors) have EST = 0. When an activity has multiple predecessors, we take the maximum — because the activity cannot begin until all its predecessors are finished.

The backward pass calculates the latest finish time (LFT) for each activity. We work right to left:

LFT(activity) = min{LFT(successor) − duration(successor)} for all successors

The final activities have LFT = project duration (the largest EST + duration found in the forward pass). When an activity has multiple successors, we take the minimum — the activity must finish early enough to allow the most time-constrained successor to start on schedule.

Worked Example (House Project)

Forward pass:
  A: EST = 0, finishes at 0 + 5 = 5
  B: EST = 5, finishes at 5 + 8 = 13
  H: EST = 5, finishes at 5 + 5 = 10
  C: EST = 13, finishes at 13 + 6 = 19
  D: EST = 13, finishes at 13 + 4 = 17
  E: EST = 13, finishes at 13 + 3 = 16
  F: EST = max(19, 17, 16) = 19, finishes at 19 + 4 = 23
  G: EST = 23, finishes at 23 + 3 = 26
Project duration: 26 days
Backward pass (LFT = 26):
  G: LFT = 26
  F: LFT = 26 − 3 = 23
  C: LFT = 23 − 4 = 19
  D: LFT = 23 − 4 = 19
  E: LFT = 23 − 4 = 19
  B: LFT = min(19 − 6, 19 − 4, 19 − 3) = min(13, 15, 16) = 13
  H: LFT = 26
  A: LFT = min(13 − 8, 26 − 5) = min(5, 21) = 5
The forward pass answers “how soon can each activity start?” The backward pass answers “how late can each activity finish without delaying the project?” Together, they reveal which activities have flexibility and which do not.

4. Float & the Critical Path

Identifying Critical Activities

The total float of an activity measures how much it can be delayed without affecting the overall project duration:

Total Float = LFT − EST − Duration

Activities with zero float are critical — they cannot be delayed at all. The sequence of critical activities from start to finish forms the critical path.

Landscaping (H) has a float of 16 days — it could start any time between day 5 and day 21 without delaying the project. This is valuable information: resources allocated to H can be temporarily redeployed to critical activities if needed. Plumbing (D) and wiring (E) have small floats of 2 and 3 days respectively, so they need closer monitoring.

Example 1: Software Launch

A software team plans a product launch with these activities:

ActivityDescriptionDuration (weeks)Depends on
PRequirements gathering3—
QUI design4P
RBackend development6P
SFrontend development5Q
TIntegration testing3R, S
UUser acceptance testing2T
▶ Show full solution

Forward pass:

  P: EST = 0, finishes at 3
  Q: EST = 3, finishes at 7
  R: EST = 3, finishes at 9
  S: EST = 7, finishes at 12
  T: EST = max(9, 12) = 12, finishes at 15
  U: EST = 15, finishes at 17

Backward pass (LFT = 17):

  U: LFT = 17
  T: LFT = 17 − 2 = 15
  R: LFT = 15 − 3 = 12
  S: LFT = 15 − 3 = 12
  Q: LFT = 12 − 5 = 7
  P: LFT = min(7 − 4, 12 − 6) = min(3, 6) = 3

Float:

  P: 3 − 0 − 3 = 0 (critical)
  Q: 7 − 3 − 4 = 0 (critical)
  R: 12 − 3 − 6 = 3
  S: 12 − 7 − 5 = 0 (critical)
  T: 15 − 12 − 3 = 0 (critical)
  U: 17 − 15 − 2 = 0 (critical)

Critical path: P → Q → S → T → U — minimum duration 17 weeks.

Backend development (R) has 3 weeks of float, so it could be delayed without affecting the launch — but only if no other delays occur.

  N: LFT = 13
  L: LFT = 13 − 3 = 10
  K: LFT = 10 − 3 = 7

  J: 2 − 0 − 2 = 0 (critical)

  N: 13 − 10 − 3 = 0 (critical)
  O: 13 − 2 − 2 = 9

Critical path: J → K → L → N — minimum duration 13 days.

Catering (M) and AV setup (O) both have substantial float. The bottleneck is the speaker → programme → print chain.
