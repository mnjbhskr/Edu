Positional Encoding

Order is not free — how Transformers learn where words stand

A Transformer sees every token at once — it has no built-in notion of first, second, or last. Without positional encoding, "dog bites man" and "man bites dog" would produce identical representations. The solution is elegant: add a unique, smooth signal to each token that encodes its position using sine and cosine waves at different frequencies, much like how a clock encodes time across hours, minutes, and seconds.

Unlike RNNs which read left-to-right, Transformers see all tokens at once. Without positional encoding, "dog bites man" and "man bites dog" would be mathematically identical — the attention mechanism treats inputs as an unordered set.

Position gives meaning — the same words in different order tell a completely different story.

PE(pos, 2i) = sin(pos / 100002i/d)

where pos = position, i = dimension index, d = model dimension
PE(pos + k) = Tk × PE(pos)     where Tk is a rotation matrix
The sinusoidal encoding is elegant: each dimension oscillates at a different frequency. Like a binary number system, this guarantees every position has a unique, smooth encoding. Dimension 0 oscillates rapidly (high frequency), while dimension d oscillates very slowly (low frequency) — together they create a unique fingerprint for every position.
The elegant trick: for any fixed offset k, the encoding PE(pos+k) can be expressed as a linear transformation of PE(pos). This means relative positions are computable via simple matrix operations — the model can learn “three words apart” without knowing absolute position. This is why sinusoidal encoding works: it encodes both absolute AND relative position simultaneously.

Sine waves at geometrically-spaced frequencies give every position a unique, smooth encoding that generalises to unseen sequence lengths.

Dot product similarity reveals the distance structure baked into sinusoidal encodings — nearby positions are naturally more similar.

Why This Matters

Positional encoding is what gives Transformers their sense of order. Without it, self-attention would treat language as a bag of words — losing syntax, grammar, and meaning. The sinusoidal design from the original "Attention Is All You Need" paper (Vaswani et al., 2017) remains widely used, though modern models like RoFormer (Su et al., 2021) have introduced Rotary Position Embeddings (RoPE) that encode relative positions through rotation matrices.

GPT-4 and similar models process sequences of thousands of tokens. Every single token carries its positional encoding, ensuring the model knows not just what each token is, but where it stands in the sequence.

Modern Alternative: Rotary Position Embedding (RoPE)

Modern models (LLaMA, GPT-NeoX, Mistral) use Rotary Position Embedding (RoPE) instead of additive sinusoidal encoding. The key idea: multiply query and key vectors by a rotation matrix that depends on position. Two tokens at positions i and j have their attention score depend only on (i − j), not on absolute position. Geometrically, RoPE rotates vectors in the complex plane — pairs of embedding dimensions form 2D rotation planes, each at a different frequency. The result preserves the distance-based attention that makes sinusoidal encoding elegant, while being naturally compatible with relative position.
