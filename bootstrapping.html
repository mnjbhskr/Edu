Bootstrapping & Resampling Methods

●●●● Intermediate

Pull yourself up by your own straps — estimating uncertainty from a single sample

1. The Bootstrap Idea

In 1979, Bradley Efron introduced a startlingly simple idea: if you have a single sample and want to know how variable your statistic is, treat your sample as if it were the population. Draw new samples from it — with replacement — compute your statistic each time, and build a distribution.

This is the bootstrap. The name comes from the phrase “pulling yourself up by your own bootstraps” — the method creates something from (almost) nothing. No assumptions about the shape of the population distribution are needed. No closed-form formula for the standard error is required.

The bootstrap works because the empirical distribution function of a sample converges to the true distribution function as sample size grows. Your sample is the best available approximation of the population — so resample from it.

Traditional statistics often requires you to assume normality or derive complex formulae for standard errors. The bootstrap sidesteps both: it lets the data speak for itself. This makes it one of the most widely used tools in modern applied statistics.

2. How It Works

The Bootstrap Algorithm

The procedure is remarkably straightforward. Given an original sample of size n:

StepActionDetail
1 Start with your sample Observed data: x1, x2, …, xn
2 Draw a bootstrap sample Sample n values with replacement from the original data
3 Compute the statistic Calculate your statistic of interest (mean, median, std dev, etc.)
4 Repeat B times Typically B = 1,000 to 10,000 repetitions
5 Analyse the distribution The B bootstrap statistics estimate the sampling distribution
Bootstrap standard error:   SEboot = √[ (1/B) ∑ (θ*b − θ̄*)2 ]
where θ*b = statistic from the b-th bootstrap sample, θ̄* = mean of all bootstrap statistics
“With replacement” is crucial. Each bootstrap sample will contain some original observations more than once and omit others entirely. On average, about 63.2% of the original observations appear in any single bootstrap sample (this follows from 1 − (1 − 1/n)n ≈ 1 − 1/e).

4. Bootstrap Confidence Intervals

Three Methods

Once you have the bootstrap distribution, there are several ways to construct a confidence interval. Each makes slightly different assumptions about the relationship between the bootstrap distribution and the true sampling distribution.

(a) Percentile Method

CIpercentile = [ θ*(0.025) , θ*(0.975) ]
Simply take the 2.5th and 97.5th percentiles of the bootstrap statistics.

The most intuitive method. It works well when the bootstrap distribution is approximately unbiased and symmetric.

(b) Basic (Pivotal) Method

CIbasic = [ 2θ̂ − θ*(0.975) , 2θ̂ − θ*(0.025) ]
where θ̂ is the statistic from the original sample.

Reflects the bootstrap distribution about the original estimate. This corrects for bias in the bootstrap distribution relative to the sample statistic.

(c) BCa (Bias-Corrected and Accelerated)

The BCa method adjusts the percentile endpoints to correct for both bias (the bootstrap distribution may not be centred on θ̂) and skewness (the distribution may be asymmetric). It requires computing a bias-correction factor ẑ0 and an acceleration factor â, making it more computationally involved but generally more accurate.

For symmetric distributions and large samples, the three methods give nearly identical results. The differences become noticeable with skewed data or small sample sizes — precisely the situations where careful interval construction matters most.

5. When Bootstrap Works & When It Doesn’t

Strengths and Limitations

Works WellCan Fail
Smooth statistics — means, medians, regression coefficients, correlation Extreme-value statistics — max, min, range; the bootstrap cannot generate values beyond the observed sample
Moderate to large samples — n ≥ 20 for most statistics Very small samples — n < 10 gives too few distinct bootstrap samples
Unknown population distribution — no normality assumption needed Heavy-tailed data with infinite variance — the empirical distribution underestimates tail behaviour
Complex statistics — ratios, differences, things with no closed-form SE Dependent data — time series, spatial data (requires block bootstrap variants)
▶ Why does the bootstrap fail for extreme values?

Consider estimating the population maximum. Your sample maximum is the largest value you observed. Every bootstrap sample draws from the original data, so the bootstrap maximum can never exceed the sample maximum. The bootstrap distribution of the maximum will be a point mass at the sample max (or occasionally below it), giving a standard error of essentially zero — which is clearly wrong.

More formally, the bootstrap works when the statistic is a smooth functional of the empirical distribution. The maximum is not smooth — it depends on a single extreme observation — so the bootstrap approximation breaks down.

6. Worked Example — Bootstrapping the Median

15 Exam Scores

A class of 15 students sits an exam. Their scores are:

{ 63, 68, 70, 72, 76, 78, 79, 82, 84, 85, 87, 88, 90, 91, 95 }
Original median: 82 (the 8th value when sorted)

Three Example Bootstrap Samples

Sample 1: { 72, 85, 63, 85, 90, 78, 82, 95, 70, 82, 91, 76, 82, 68, 88 } → median = 82
Sample 2: { 91, 70, 84, 63, 78, 90, 95, 72, 87, 70, 76, 84, 68, 91, 79 } → median = 79
Sample 3: { 88, 95, 85, 88, 72, 91, 84, 78, 90, 85, 87, 76, 95, 82, 91 } → median = 87

After B = 2,000 bootstrap samples, the distribution of bootstrap medians gives:

Bootstrap SE
4.78
95% CI (Percentile)
[ 72, 90 ]
Original Median
82
Compare this with the naive approach: there is no simple closed-form formula for the standard error of the median. The bootstrap gives us a practical answer — the median of exam scores is approximately 82 ± 4.8, with a 95% confidence interval of roughly [72, 90].
